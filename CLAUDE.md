# LeArchive — project conventions for Claude

Local overrides and reminders for anyone (human or agent) working on this repo.

## Repo map

- `scraper/` — Python + Poetry package `le_archive`. Scraper + enrichment workers.
- `site/` — Vite + React + TS SPA with InstantSearch.
- `.env` at repo root holds Algolia creds. `load_dotenv(..., override=True)` — shell exports `ALGOLIA_*` would otherwise shadow them.

## Before handing off frontend work (mandatory)

1. `cd site && npm run lint` — must pass with **zero** warnings (`--max-warnings 0`).
2. `npm run build` — now runs lint first, so green build proves green lint.
3. `npm run test:e2e` — Playwright smoke tests under `site/e2e/`. Takes ~6s. Covers: boot, play-button interaction (React hook-order regression witness), favorites star, `/` keyboard shortcut.
4. If you add a new interactive component or wire a new global event, add a matching spec in `site/e2e/`. Runtime-only bugs (hooks order, uncaught promise rejection, blank screen) don't show up in `tsc` or `eslint` — the spec suite is how they get caught.

## React hooks

- **Never call a hook after an early return.** All `useState` / `useEffect` / `useRef` / `useMemo` / custom hooks must be in the top block of the component, before any `if (!x) return null;`. `eslint-plugin-react-hooks` catches it — but only if lint actually runs. See guardrail above.
- **Never write to a ref during render.** `myRef.current = value` at the top level of the component body is a `react-hooks/refs` violation. Sync props to refs inside `useEffect`.
- `setState` inside an effect is usually a smell. If you genuinely need to (e.g. resetting on an input change, hydrating from a cache), the rule is `react-hooks/set-state-in-effect`. Either refactor to derived state or disable with a comment explaining *why* it's safe here.

## Dev practices

- **Never ship code you haven't at least lint-checked.** The user will not run lint for you. If you edited `site/src`, you run `npm run lint` before declaring the change done. If it fails, fix first — a dev overlay stacktrace that reaches the user is a regression.
- **Add a test for every runtime bug you fix.** If a bug made it into the browser console, the smoke suite didn't catch it; fix the hole. One-line test > one-line fix.
- **No AI fingerprints in commits or code.** No `Co-Authored-By: Claude`, no "generated by" footers. The user is the sole author; tooling is invisible.
- `rg` and `sg` are available for navigation. Prefer them over `grep`/`find`.
- Python: `poetry` only. No `pip install` in the venv.
- P6 audio worker runs in the background via `nohup`, **sharded** into 2+ cooperating processes. Each worker takes `objectID` hashes modulo `N`. Commands:
  ```
  nohup poetry run python -m le_archive.enrich_audio --shard 0/2 > /tmp/learchive-p6-a.log 2>&1 &
  nohup poetry run python -m le_archive.enrich_audio --shard 1/2 > /tmp/learchive-p6-b.log 2>&1 &
  ```
  Check: `pgrep -af "enrich_audio --shard"`. Progress: `tail -3 /tmp/learchive-p6-a.log`.
- **Sharded workers MUST NOT rewrite raw_sets.json** — two writers race and clobber each other. Sharded code path detects this (`shard_n > 1 → write_raw=False`). Algolia is authoritative; `poetry run python -m le_archive.index` resyncs raw_sets post-run.
- **Atomic saves**: every worker uses `le_archive._io.atomic_write_json` (tempfile + `os.replace` + fsync). A crashed mid-write cannot leave the file torn. Single-worker runs checkpoint **after every set** (`CHECKPOINT_EVERY=1`).
- **yt-dlp no-transcode**: do **not** pass `--extract-audio --audio-format mp3`. librosa decodes webm/opus natively via audioread + ffmpeg; transcoding is real-time-bound and costs ~10× per set for no gain.
- **Mixcloud rate-limits yt-dlp at scale.** 2026-04-22 P6 run: after ~560 successful downloads in ~14h, yt-dlp started failing en-masse on the tail (329 sets skipped). Symptom in logs: per-set rate crashes from ~180s to ~1s/set — download failures zipping through, not miracles. Mitigation for batch runs: sequential (not sharded), `--delay 5+` between requests, plan for a 24h quota window. Mark failures explicitly with `_enrichment.audio_failed=true` so retries don't re-hit terminally-dead URLs.
- **Ubuntu 24.04 apt package for Chromaprint fpcalc is `libchromaprint-tools`** (NOT `chromaprint-tools` — that's the old Debian name). P9 `enrich_tracks.py` requires this installed.

## Algolia index

- App `TDNMRH8LS3`, index `archive_sets`.
- Faceted numerics: `bpm`, `brightness`, `noisiness`, `energy_mean`, `energy_dynamic_range`.
- Boolean gates: `_enrichment.{mixcloud,artists,mood,audio}` — query `_enrichment.audio:true` for P6 coverage.
- Record cap bumped to 100KB for fingerprints (user is an Algolia employee, no bill).
- `algolia_client.py` is the single source of client truth; import it, don't reconstruct.

## Source site quirks

- `hetarchief.deschoolamsterdam.nl` is ProcessWire + HTMX + Alpine.
- Mixcloud iframes are AJAX-loaded via `/sets/{slug}/embed/` — scraper fetches that fragment to extract the real URL. Mixcloud slug ≠ archive slug (event names sometimes differ).
- Some sets have no Mixcloud release (unpublished). Keep the record, mark `mixcloud_url=null`; the Hit row shows `—` instead of a play button.
